# 需求文档（Requirements）

## 1. 项目简介

**德州扑克记分工具**：面向小范围牌友/朋友局的轻量记分网页应用。前端纯静态，可直接用浏览器打开；可选启动后端服务，按「工作区 ID」把数据存到服务器磁盘，便于多设备共用与备份。

## 2. 目标用户与场景

| 用户 | 场景 |
|------|------|
| 牌友群组发起人 | 录入每局每人的输赢分数，赛后查看排行 |
| 多设备使用的玩家 | 在家用电脑录入，赛后用手机查看排行 |
| 仅本地使用者 | 不开后端，全部数据存在本机浏览器（IndexedDB） |

## 3. 功能需求

### 3.1 成员管理
- 添加成员（名称唯一，最长 20 字符）
- 删除成员（已有积分记录的成员不允许删除）
- 列表展示与计数

### 3.2 记分录入
- 一次录入多名成员的得分（留空表示该成员本局不参与）
- 支持自定义时间（日期 + 时间）与备注（局号 / 场地）
- 同一时间点提交的记录视为同一局

### 3.3 历史记录
- 时间倒序展示，可按成员筛选
- 支持编辑（成员、积分、时间、备注）
- 支持删除（带确认弹窗）

### 3.4 排行榜
- 时间维度：总榜 / 今日 / 本周 / 本月 / 近 7 天 / 近 30 天 / 自定义日期范围
- 可切换「显示总积分对比」列
- 按积分降序，并标注名次

### 3.5 积分明细（图表）
- 多选成员，绘制折线图
- 模式：按次（每局得分）/ 累计（累计积分）
- 含汇总信息（总场次、最大值、最小值等）

### 3.6 数据管理
- 导出 / 导入 JSON 备份
- 导出 CSV
- 一键清空所有本地数据

### 3.7 云端同步（可选）
- 配置后端服务地址 + 工作区 ID
- 测试连接（健康检查）
- 上传到服务器（整包覆盖）
- 从服务器拉取（整包覆盖本地）
- 自动上传：本地变更后约 0.8s 防抖上传

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 兼容性 | 支持现代浏览器（Chrome/Edge/Safari 最近 2 年版本） |
| 性能 | 千条以内记录交互无卡顿；图表懒渲染 |
| 隐私 | 默认数据仅本机；后端由用户自管，不内置鉴权（弱口令模式） |
| 离线可用 | 不依赖网络也可记分（仅本地模式） |
| 国际化 | 简体中文界面 |

## 5. 数据模型

```jsonc
// member
{ "id": "string", "name": "string", "createdAt": 1714300000000 }

// record
{
  "id": "string",
  "memberId": "string",
  "score": 0,            // 数字，可为负
  "time": 1714300000000, // ms
  "note": "string"       // 可空
}

// 工作区整包（与 IndexedDB 等价）
{ "members": [...], "records": [...], "updatedAt": 1714300000000 }
```

## 6. 接口需求（后端）

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/workspaces/:id` | 读整包 |
| PUT | `/api/workspaces/:id` | 整包覆盖保存 |

工作区 ID 校验：`^[a-zA-Z0-9_-]{8,128}$`，防止路径穿越。

## 7. 验收标准（节选）

- [ ] 添加成员后能在录入页选中
- [ ] 同一时间录入的多个成员视为同一局
- [ ] 编辑/删除后排行榜与图表立即同步刷新
- [ ] 关闭后端时，前端在「测试连接」给出友好错误，不影响本地使用
- [ ] 工作区 ID 不合法时后端返回 400

## 8. 不在范围内（Out of Scope）

- 用户登录 / 多租户鉴权
- 增量同步、冲突解决（当前为整包覆盖、末次写入为准）
- 移动端 App
- 抽水、佣金、复杂结算规则
