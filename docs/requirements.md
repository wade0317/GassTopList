# 需求文档（Requirements）

## 1. 项目简介

**德州扑克记分工具**：面向小范围牌友/朋友局的轻量记分网页应用。前端页面由后端托管，业务数据统一存到服务端 SQLite 数据库，浏览器不再本地持久化业务数据。

## 2. 目标用户与场景

| 用户 | 场景 |
|------|------|
| 牌友群组发起人 | 录入每局每人的输赢分数，赛后查看排行 |
| 多设备使用的玩家 | 在家用电脑录入，赛后用手机查看排行 |
| 组织者 / 记分员 | 在同一个账本里集中维护成员与战绩 |

## 3. 功能需求

### 3.1 成员管理
- 添加成员（名称唯一，最长 20 字符）
- 删除成员（已有积分记录的成员不允许删除）
- 列表展示与计数

### 3.2 记分录入
- 一次录入多名成员的得分（留空表示该成员本局不参与）
- 支持自定义时间（日期 + 时间）与备注（局号 / 场地）
- 同一时间点提交的记录视为同一局

### 3.3 历史记录
- 时间倒序展示，可按成员筛选
- 支持编辑（成员、积分、时间、备注）
- 支持删除（带确认弹窗）

### 3.4 排行榜
- 时间维度：总榜 / 今日 / 本周 / 本月 / 近 7 天 / 近 30 天 / 自定义日期范围
- 可切换「显示总积分对比」列
- 按积分降序，并标注名次

### 3.5 积分明细（图表）
- 多选成员，绘制折线图
- 模式：按次（每局得分）/ 累计（累计积分）
- 含汇总信息（总场次、最大值、最小值等）

### 3.6 数据持久化
- 页面不再提供导入 / 导出 / 手动同步入口
- 所有成员与积分记录直接存入后端 SQLite
- 录入积分、成员变更、记录编辑删除后立即写入数据库
- 页面刷新后自动从数据库恢复当前默认账本

### 3.7 登录与权限
- 页面启动后先显示登录界面
- 提供两个入口：**游客登录**、**管理员登录**
- 游客可直接登录，无需密码
- 管理员登录必须输入密码
- 游客登录后仅可查看数据，不能录入、编辑、删除或新增成员
- 管理员登录后可查看并执行全部录入与管理操作

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 兼容性 | 支持现代浏览器（Chrome/Edge/Safari 最近 2 年版本） |
| 性能 | 千条以内记录交互无卡顿；图表懒渲染 |
| 隐私 | 数据保存在用户自管后端；采用游客 / 管理员两级权限 |
| 离线可用 | 不支持离线记分，依赖后端服务可用 |
| 国际化 | 简体中文界面 |

## 5. 数据模型

```jsonc
// member
{ "id": "string", "name": "string", "createdAt": 1714300000000 }

// record
{
  "id": "string",
  "memberId": "string",
  "score": 0,            // 数字，可为负
  "time": 1714300000000, // ms
  "note": "string"       // 可空
}

// 默认账本快照（服务端返回结构）
{ "members": [...], "records": [...], "updatedAt": 1714300000000 }
```

## 6. 接口需求（后端）

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/auth/me` | 查询当前登录角色 |
| POST | `/api/auth/guest` | 游客登录 |
| POST | `/api/auth/admin` | 管理员登录 |
| POST | `/api/auth/logout` | 退出登录 |
| GET | `/api/state` | 读取默认账本 |
| POST | `/api/members` | 新增成员 |
| DELETE | `/api/members/:id` | 删除成员 |
| POST | `/api/records` | 批量新增积分记录 |
| PUT | `/api/records/:id` | 更新单条积分记录 |
| DELETE | `/api/records/:id` | 删除单条积分记录 |
| DELETE | `/api/state` | 清空默认账本 |

## 7. 验收标准（节选）

- [ ] 添加成员后能在录入页选中
- [ ] 同一时间录入的多个成员视为同一局
- [ ] 编辑/删除后排行榜与图表立即同步刷新
- [ ] 页面刷新后可从数据库恢复已有成员与积分记录
- [ ] 后端未启动时，前端给出明确错误提示
- [ ] 游客登录后无法录入、编辑、删除数据
- [ ] 管理员登录后可正常录入和修改数据

## 8. 不在范围内（Out of Scope）

- 多账本 / 多工作区切换
- 移动端 App
- 抽水、佣金、复杂结算规则
